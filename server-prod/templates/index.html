📖 About Smart Garden

Smart Garden is a solar-powered, AI-driven irrigation system built from scratch in Duvall, WA. It uses an ESP32 microcontroller to actuate DC latching solenoid valves via L298N H-bridges, controlled by a Python server running on a home Acer mini-PC. Weather data from Open-Meteo drives intelligent skip/water decisions using FAO Penman-Monteith ET₀ reference evapotranspiration.

🌐 9 Zones📡 4 Soil Sensors🌤️ Open-Meteo Weather ☀️ ET₀ FAO Penman-Monteith💰 Duvall Billing Tiers

🏗️ How It Works

The system has three layers: hardware (ESP32 + sensors + valves), a Python server (decision engine + database + web dashboard), and cloud weather data (Open-Meteo API). Every 15 minutes, the server evaluates each zone and decides: water, skip, or wait.

Decision Cycle (every 15 minutes)

  1. Fetch ESP32 status — soil moisture (raw ADC + calibrated %), temperature, humidity, valve states
  2. Fetch Open-Meteo weather — current conditions + 24-hour rain forecast + ET₀
  3. Calculate weather scale — Zimmerman method adjusts runtime (0% skip → 200% double)
  4. For each zone: check skip rules (rain, wind, freeze, soil wet, budget)
  5. If not skipped and soil < dry_trigger: open valve for scaled runtime, log soil_before
  6. On valve close: log soil_after, duration, gallons used, ET demand, trigger reason
  7. If skipped: log skip reason + full weather conditions snapshot (JSON)
  8. Update daily summaries — aggregate gallons, savings, cost avoidance

🗺️ Intelligence Roadmap

✅ Phase 1 — Weather-Aware Skip Logic COMPLETE

Integrated Open-Meteo weather API with ET₀ (FAO Penman-Monteith), rain forecast, wind speed, and freeze temperature into skip/water decisions. Every zone has seasonal Kc crop coefficients. Billing tier awareness prevents over-watering when rates are high.

Skip rules: Rain ≥5mm + ≥60% prob · Recent rain ≥8mm · Wind >15 mph (sprinklers) · Freeze <35°F · Budget tightening
🔧 Phase 2 — Zimmerman Weather-Adjusted Runtimes ACTIVE

Instead of fixed runtimes, scale watering duration by a weather factor. Hot dry days water longer; cool humid days water less. Based on the Zimmerman method used by Rachio and OpenSprinkler, calibrated for Duvall WA's cooler climate (baseline 60°F / 50% humidity vs the standard 70°F / 30%).

Formula: scale = 100 + (baseline_humidity − humidity) + (temp − baseline_temp) × 4 + rain_24h × −200
Range: 0% (skip) to 200% (double runtime) · Sources: OpenSprinkler Zimmerman, Rachio Weather Intelligence Plus
🔮 Phase 3 — Soil Water Balance (Checkbook Method) PLANNED

Track a running soil water balance per zone: subtract daily ET₀×Kc (evaporation), add rain and irrigation. Water when the balance drops below Management Allowed Depletion (MAD). This replaces reactive sensor-only triggering with predictive scheduling — the system knows when to water before the soil gets dry.

Method: NDSU Checkbook / UF/IFAS ET scheduling
Balance: daily_balance -= ET₀ × Kc · daily_balance += rain + irrigation
Sources: NDSU Extension Service, UF/IFAS, FAO-56

🌡️🌱 Soil Temperature (DS18B20) NEEDS HARDWARE
Waterproof DS18B20 probe buried at sensor depth. Enables temperature-compensated soil moisture readings (capacitive sensors drift ~5-10% between summer/winter at the same actual moisture). Also detects frost risk for root zone protection.
Buy: DS18B20 waterproof probe (~$3) · Wire: 1-Wire protocol, single GPIO + 4.7kΩ pull-up
🌡️
No soil temperature data yet — sensor not connected
🌀 Barometric Pressure (BME280) NEEDS HARDWARE
Local barometric pressure for rain prediction (falling pressure = incoming weather). Rapid drops (>2 hPa/hr) can trigger pre-emptive irrigation skips. Also provides precision temperature + humidity as a cross-check against DHT22 and weather API.
Buy: BME280 breakout (~$5) · Wire: I2C (SDA + SCL), shares bus with other I2C sensors
🌀
No pressure data yet — sensor not connected
☀️🔋 Solar Panel Power (INA219) NEEDS HARDWARE
Voltage + current from the solar panel. Know if the panel is generating enough power, detect shading or degradation, and correlate charging current with ESP32 uptime/reboots. Dual-axis: voltage (V) left, current (mA) right.
Buy: INA219 breakout (~$3) · Wire: I2C + in-line with solar panel positive lead
☀️
No solar panel data yet — sensor not connected
💧📏 Water Flow (YF-S201) NEEDS HARDWARE
Hall-effect flow sensor inline with your irrigation supply. Measures actual gallons per watering cycle instead of estimating from valve-open time × GPM. Enables leak detection (flow when no valve is open) and accurate water billing.
Buy: YF-S201 flow sensor (~$8) · Wire: pulse output to GPIO (interrupt-driven counting)
