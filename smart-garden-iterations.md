## 🔁 Recurring mistake patterns — read this first

These are not one-time bugs. They're patterns that have repeated across sessions. Each row is a meta-pattern, not a single iteration.

| # | Pattern | First seen | Repeated | Root cause | Guard |
|---|---------|------------|----------|------------|-------|
| **R1** | "This is the last flash — you can box it up" said with high confidence after testing in the wrong environment | 2026-04-21 (×4 in one session) | 2026-04-26 (declared MOSFET work done while USB still plugged in → wedged on battery) | Conflating "passes the test I happened to run" with "production-ready." Treating USB-tethered metrics as proof for solar/wall behavior. | The "before saying it's fixed" gate: USB unplugged → real power source → deployed location → A+B health probes 5 min clean → THEN claim. | **R2** | Premature fix declaration on intermittent bugs after one passing test | 2026-04-26 23:45 (V4 socket reset declared FIXED after 7/7 short-gap test, then 120s gap failed minutes later) | Likely earlier instances buried in archive | Stopping at first green light instead of running soak test that exercises the full failure-mode space (longer gaps, longer durations, real load). | For intermittent bugs: minimum 10-minute soak with worst-case gap intervals (60s/120s/180s) before declaring fixed. |
| **R3** | Brownout from Wanderer load output during high-current bursts | 2026-04-05 (initial buck crash-loop) | 2026-04-21 (×4 OTA brick attempts), 2026-04-26 (suspected post-USB-unplug wedge) | LM2596 + Wanderer load output sags during WiFi TX bursts / OTA / multi-valve pulses. ## 📚 How to keep this doc honest

After every session that changes anything on the chip or server:

1. Add one row per iteration to the chronological table (Date, Category, Goal, What tried, Result, Lesson).
2. If the result is `❌` or `⚠️`, look at the recurring patterns table at the top — does this match an existing R-pattern? If yes, increment your awareness; if it's a brand new pattern that shows up twice, add a new R-row.
3. If you said "this is fixed" and it's not — it goes in BOTH places (the iteration row AND a recurring pattern instance).
4. Don't delete failed iterations. The point of this doc is to make the failures visible so they stop repeating.
